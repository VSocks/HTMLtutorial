Intrudoção básica a repositórios e controle de versões

O que é git?

Git é um sistema de controle de versões, ou seja, um software utilizado para desenvolver programas que oferece um aplo controle sobre cada mudança a ser feita no código. Desse modo, é possível que diversas pessoas trabalhem sobre o mesmo projeto e ao mesmo tempo e ultimamente juntem seus trabalhos em um só, e que cada erro que seja cometido durante a programação possa ser revertido para uma versão anterior, entre outras aplicações.

O que é github?

Github é o repositório do git, ou seja, é uma plataforma em nuvem em que cada pessoa possui seu perfil onde podem armazenar seus projetos e colaborar com outros desenvolvedores.

Como instalar o git e criar conta no github?

O git pode ser baixado pelo site do git. Caso esteja utilizando alguma versão do linux, também pode ser feito pelo terminal. Por exemplo, no Ubuntu ele pode ser instalado com os seguintes comandos:

  1. sudo apt update
  2. sudo apt upgrade
  3. sudo apt install git
Finalizado a instalação, confere-se a versão com git --version e define-se o nome de usuário e-mail da conta do github com git config --global user.name "nome" e git config --global user.email "email". Para criar a conta no github o processo é o mesmo que em qualquer outro site: o usuário escolhe um nome e um e-mail, o site verifica se ele não é um robô, e então o processo está completo. Caso seja estudante, pode estar entitulado a uma versão prêmium do site.

Alguns comandos do git

init

Inicia o git, criando um novo repositório local vazio para o projeto

clone

Cria uma cópia de um repositório já existente do github.

branch

Usado para listar os existentes "ramos", isto é, diferentes ambientes de programação dentro do mesmo repositório, também como para criar novos ramos ou deleta-los. Um conjunto de ramos se chama uma árvore.

checkout

Usado para navegar entre ramos ou para restaurar ramos existentes.

status

Mostra o estado atual do arquivo atual e da área de "staging."

diff

Exibe diferenças entre commits, ramos e diretórios.

add

Envia arquivos para a área de "staging", preparando para que as mudanças possam receber "commit."

commit

Salva o estado atual dos arquivos no estado de "staging", aplicando de fato suas alterações.

pull

Baixa um ramo de um repositório do github e merge com o ramo atual.

revert

Desfaz um commit.

merge

Mescla dois ou mais arquivos.

stash

Salva todas as mudanças feitas nos arquivos, sem dar commit.

rm

Remove arquivos do repositório ou ramos da árvore.

config

Define configurações para a instalação do git, tipicamente só usando no download.

reset

Desfaz mudanças feitas no diretório atual.

remote

Usado para criar uma conexão entre o repositório online do github e o repositório local do computador.

fetch

Baixa um ramo e os arquivos de um repositório do github, mas não os integra com nada do repositório atual.

show

Mostra os elementos especificados, como ramos, commits, e tags.

help

Exibe documentação sobre o git.

rebase

Permite movimentar ramos, para evitar conflitos na hora de junta-los.

blame

Exibe quem foi a última pessoa a modificar cada linha do arquivo.

tag

Usado para marcar pontos importantes no histórico do repositório, ou seja, definir "versões" do código.
